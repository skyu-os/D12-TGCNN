## 核心算法：A* 搜索

### 算法原理

A* 是一种启发式搜索算法，结合了 Dijkstra 的最优性和贪心搜索的效率。

**核心公式**:
```
f(n) = g(n) + h(n)
```

其中：
- **g(n)**: 从起点到节点 n 的实际代价
- **h(n)**: 从节点 n 到终点的启发式估计（直线距离）
- **f(n)**: 节点 n 的总评估代价

### 算法流程

```python
# 伪代码
def astar_search(start, goal):
    open_set = PriorityQueue()  # 最小堆，按 f(n) 排序
    open_set.put((0, start))
    
    g_score = {start: 0}  # 实际代价
    f_score = {start: h(start, goal)}  # 评估代价
    came_from = {}  # 路径重建
    
    while not open_set.empty():
        current = open_set.get()[1]  # 取 f 最小的节点
        
        if current == goal:
            return reconstruct_path(came_from, current)
        
        for neighbor in neighbors(current):
            # 边松弛操作
            tentative_g = g_score[current] + edge_weight(current, neighbor)
            
            if tentative_g < g_score.get(neighbor, ∞):
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g
                f = tentative_g + h(neighbor, goal)
                f_score[neighbor] = f
                open_set.put((f, neighbor))
    
    return None  # 不可达
```

### 启发函数：Haversine 公式

**代码位置**: `backend/routing/astar.py` 第 22-33 行

**公式**:
```python
R = 6371000  # 地球半径（米）
dlat = radians(lat2 - lat1)
dlon = radians(lon2 - lon1)

a = sin²(dlat/2) + cos(lat1) × cos(lat2) × sin²(dlon/2)
c = 2 × atan2(√a, √(1-a))
distance = R × c  # 返回单位：米
```

**特点**:
- ✅ 计算球面两点间的最短距离（大圆距离）
- ✅ 满足 A* 的可采纳性（h(n) ≤ 实际距离）
- ✅ 满足一致性（单调性）

### 边权重选择

## 时间计算详解

### 计算位置

**文件**: `backend/graph/road_graph.py`  
**方法**: `get_edge_weight()` 第 118-130 行

### 核心公式

```
时间 = 距离 / 速度

time_s = Σ (length_m / speed_m_s)
```

### 计算流程

```python
def get_edge_weight(self, u, v, key=0, weight_type="time"):
    length = get_edge_length(u, v, key)  # 米
    
    if weight_type == "distance":
        return length
    
    # 计算时间权重
    speed_kmh = get_edge_speed(u, v, key)  # km/h
    speed_m_s = speed_kmh * 1000 / 3600   # 转换为米/秒
    
    time_s = length / speed_m_s if speed_m_s > 0 else float('inf')
    return time_s
```

### 速度获取逻辑

**代码位置**: `backend/graph/road_graph.py` 第 109-116 行

```python
def get_edge_speed(self, u, v, key=0):
    """获取边的限速（km/h）"""
    data = self.G.edges[u, v, key]
    speed = data.get("maxspeed", None)
    
    if speed:
        return self._parse_speed(speed)
    
    return 60  # 默认 60 km/h
```

### 速度解析器

**代码位置**: `backend/graph/road_graph.py` 第 152-184 行

**支持的格式**:
- `"55 mph"` → 88.5 km/h (×1.609)
- `"40 km/h"` → 40 km/h
- `["40 mph", "45 mph"]` → 64.4 km/h (取第一个)
- 纯数值: `60` → 60 km/h

**默认值**: 当 OSM 数据中没有 `maxspeed` 属性时，使用 **60 km/h**

### 时间累加

**代码位置**: `backend/routing/astar.py` 第 110-125 行

```python
total_time = 0

for i in range(len(path) - 1):
    u, v = path[i], path[i + 1]
```

### 计算特点

✅ **优点**:
- 考虑了不同道路的限速差异
- 自动选择时间最短的平行边

⚠️ **限制**:
- **完全静态**：假设车辆始终以限速行驶
- 不考虑拥堵、事故、施工等因素
- 不包含红绿灯等待时间
- 不考虑天气影响

## 静态数据标注

### ⚠️ 当前系统的静态特性

| 数据类型 | 来源 | 更新频率 | 局限性 |
|---------|------|---------|--------|
| **路网结构** | OSM | 手动更新 | 新建道路不包含 |
| **边长度** | OSM | 静态 | 不变 |
| **道路限速** | OSM | 静态 | 实际车速低于限速 |
| **行驶时间** | 计算得出 | 静态 | 不反映真实路况 |

### 静态假设

1. **速度假设**:
   ```
   实际速度 = min(道路限速, 60 km/h)
   ```
   - ❌ 不考虑交通拥堵
   - ❌ 不考虑高峰/平峰差异
   - ❌ 不考虑事故、施工

2. **时间假设**:
   ```
   行驶时间 = 距离 / 限速
   ```
   - ❌ 无红绿灯延误
   - ❌ 无转弯时间
   - ❌ 无停车/起步时间

3. **路径假设**:
   - OSM 路网完整准确
   - 所有道路均可通行
   - 无实时封路信息

### 对比：静态 vs 动态

| 特性 | 当前静态系统 | 理想动态系统 |
|------|------------|------------|
| 速度数据 | OSM 限速（固定） | 实时交通速度 |
| 更新频率 | 手动更新 | 实时/每5分钟 |
| 时间精度 | ±50% 误差 | ±10% 误差 |
| 拥堵感知 | ❌ 无 | ✅ 有 |
| 事故响应 | ❌ 无 | ✅ 有 |

### 当前问题

1. **时间估算不准**: 假设车辆始终以限速行驶，实际误差可达 50%
2. **无实时数据**: 不考虑拥堵、事故、施工等因素

4. **速度模型简单**: 只有默认限速，无历史平均速度

```python
def get_edge_speed(self, u, v, key=0, time_of_day=None):
```
