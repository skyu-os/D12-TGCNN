# 🚗 传感器数据覆盖解决方案

## 📊 问题分析

### 当前情况
- **路网规模**: 312,869个节点，776,851条边
- **传感器数量**: 1,290个PeMS传感器
- **覆盖率**: 约0.4% (1290/312869)
- **数据特点**: 传感器主要分布在高速公路上，普通道路缺乏数据

### 核心挑战
历史数据只有传感器位置的速度信息，但路网大部分节点没有直接传感器数据，需要通过插值方法扩展到整个路网。

---

## 🔧 解决方案架构

### 1. 传感器-路网映射器 (`sensor_road_mapper.py`)

**核心功能：**
- 将每个传感器映射到最近的路网节点
- 为每个路网节点分配最近的传感器
- 为路网边分配对应的传感器
- 评估传感器覆盖情况

**主要类：**
```python
SensorRoadMapper      # 传感器-路网映射管理器
SpeedInterpolator      # 速度插值器
```

---

## 💡 三种插值策略

### 策略1: 最近邻插值 (Nearest Neighbor)

**原理：** 每个路网边使用其最近传感器的速度

**优点：**
- 计算快速
- 实现简单
- 保持传感器原始数据

**缺点：**
- 速度场不连续
- 边界处可能突变

**适用场景：** 快速查询、实时性要求高

```python
interpolator = SpeedInterpolator(mapper)
edge_speeds = interpolator.create_speed_field(
    sensor_speeds, 
    method='nearest'
)
```

---

### 策略2: 反距离加权插值 (Inverse Distance Weighting)

**原理：** 使用多个传感器的加权平均，权重与距离成反比

**公式：**
```
v_edge = Σ(w_i × v_sensor_i) / Σ(w_i)
w_i = 1 / (distance_i + ε)
```

**优点：**
- 速度场平滑
- 考虑多个传感器影响
- 更符合实际交通流

**缺点：**
- 计算量较大
- 需要调优距离衰减参数

**适用场景：** 高精度路径规划、交通预测

```python
edge_speeds = interpolator.create_speed_field(
    sensor_speeds, 
    method='inverse_distance'
)
```

---

### 策略3: 区域平均插值 (Regional Average)

**原理：** 将路网划分为区域，每个区域使用区域内传感器的平均速度

**优点：**
- 计算效率高
- 减少噪声影响
- 适合宏观分析

**缺点：**
- 区域边界可能不连续
- 丢失局部细节

**适用场景：** 大范围路网、简化计算

```python
edge_speeds = interpolator.create_speed_field(
    sensor_speeds, 
    method='average'
)
```

```python
def hierarchical_interpolate(mapper, sensor_speeds):
    edge_speeds = {}
    
    for u, v, key in mapper.graph.G.edges(keys=True):
        edge_data = mapper.graph.G.edges[u, v, key]
        highway_type = edge_data.get('highway', '')
        
        # 主干道使用反距离加权
        if highway_type in ['primary', 'trunk', 'motorway']:
            speed = interpolator._inverse_distance_interpolate(
                u, v, key, sensor_speeds
            )
        # 普通道路使用最近邻
        else:
            speed = interpolator._nearest_interpolate(
                u, v, key, sensor_speeds
            )
            
        edge_speeds[(u, v, key)] = speed
    
    return edge_speeds
```

## 🎯 最佳实践

### 1. 插值方法选择

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 实时路径规划 | nearest | 速度快，满足实时性 |
| 交通预测分析 | inverse_distance | 精度高，考虑多源影响 |
| 大范围查询 | average | 效率高，减少计算 |
| 高速公路 | inverse_distance | 速度场平滑 |
| 城市道路 | nearest | 保持局部特性 |

### 2. 参数调优

### 3. 数据质量检查

```python
def validate_speed_field(edge_speeds):
    """验证速度场的合理性"""
    speeds = list(edge_speeds.values())
    
    # 检查范围
    assert min(speeds) > 0, "速度不能为负"
    assert max(speeds) < 150, "速度异常过高"
    
    # 检查连续性
    sorted_speeds = sorted(speeds)
    jumps = [
        sorted_speeds[i+1] - sorted_speeds[i] 
        for i in range(len(sorted_speeds)-1)
    ]
    assert max(jumps) < 30, "速度场不连续，存在突变"
    
    print("✅ 速度场验证通过")
```
