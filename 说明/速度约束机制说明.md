# 🚗 路径规划速度约束机制说明

## 📊 当前速度约束体系

### 1. 速度数据来源

路径规划系统中的速度约束主要来自两个层次：

#### **层次1: OSM基础限速** (静态约束)

**数据来源**: OpenStreetMap的 `maxspeed` 属性

**位置**: `backend/graph/road_graph.py:109-116`

```python
def get_edge_speed(self, u, v, key=0):
    """获取边的限速（km/h），无数据则返回默认值"""
    data = self.G.edges[u, v, key]
    # OSM 的 maxspeed 通常是字符串如 "55 mph"
    speed = data.get("maxspeed", None)
    if speed:
        return self._parse_speed(speed)
    return 60  # 默认 60 km/h
```

**特点**:
- ✅ 直接从OSM路网数据获取
- ✅ 每条道路都有法定限速
- ❌ 不反映实时交通状况
- ❌ 不考虑拥堵、事故等因素

**速度解析** (`road_graph.py:152-184`):
```python
@staticmethod
def _parse_speed(speed_str):
    """解析速度字符串，如 '55 mph' -> km/h"""
    # 1. 支持数值类型
    if isinstance(speed_str, (int, float)):
        return float(speed_str)
    
    # 2. 支持列表 ['40 mph', '45 mph']，取第一个
    if isinstance(speed_str, list):
        speed_str = speed_str[0]
    
    # 3. 解析英制单位
    if "mph" in speed_str:
        return float(speed_str.replace("mph", "").strip()) * 1.609
    
    # 4. 解析公制单位
    if "km/h" in speed_str or "kmph" in speed_str:
        return float(speed_str.replace("km/h", "").replace("kmph", "").strip())
    
    # 5. 默认值
    return 60
```

#### **层次2: 预测/实时速度** (动态约束)

**数据来源**: 
- PeMS传感器历史数据
- T-GCN/LSTM预测模型输出
- 实时交通数据（可选）

**集成方式**: 通过 `SensorRoadMapper` 和 `SpeedInterpolator`

**工作流程**:
```python
# 1. 获取传感器速度数据
sensor_speeds = {
    1201044: 65.5,  # 传感器ID: 实际速度
    1201052: 58.2,
}

# 2. 插值到全路网
interpolator = SpeedInterpolator(mapper)
edge_speeds = interpolator.create_speed_field(
    sensor_speeds,
    method='inverse_distance'
)

# 3. 更新路网权重
for (u, v, key), speed in edge_speeds.items():
    road_graph.G.edges[u, v, key]['predicted_speed'] = speed
```

## ⚖️ 权重计算机制

### 当前支持的权重类型

#### **类型1: distance权重** (距离优先)

**代码位置**: `road_graph.py:124-125`

```python
if weight_type == "distance":
    return length  # 直接返回道路长度（米）
```

**特点**:
- 不考虑速度
- 寻找几何最短路径
- 适合：不考虑通行时间的场景

#### **类型2: time权重** (时间优先)

**代码位置**: `road_graph.py:127-130`

```python
# time = length / speed
speed = self.get_edge_speed(u, v, key)
speed_ms = speed * 1000 / 3600  # 转换为米/秒
return length / speed_ms if speed_ms > 0 else float("inf")
```

**公式**:
```
通行时间(秒) = 道路长度(米) / 速度(米/秒)
速度(米/秒) = 限速(km/h) × 1000 / 3600
```

**示例**:
```
道路长度: 1000米
限速: 60 km/h = 16.67 m/s
通行时间: 1000 / 16.67 = 60秒
```

## 🔧 速度约束的三种模式

### 模式1: 静态限速模式 (当前默认)

**使用场景**: 基础路径规划

**速度来源**: OSM的maxspeed属性

**优点**:
- 简单直接
- 不需要额外数据
- 反映道路设计速度

**缺点**:
- 不反映实际交通状况
- 可能高估通行速度

**使用方法**:
```python
# 使用OSM限速
result = router.find_path(
    start_node, 
    end_node, 
    weight_type='time'  # 使用OSM限速计算时间
)
```

### 模式2: 预测速度模式 (推荐)

**使用场景**: 感知路径规划

**速度来源**: T-GCN/LSTM预测模型

**优点**:
- 反映预测的交通状况
- 考虑时间、天气等因素
- 更接近实际通行时间

**缺点**:
- 需要训练预测模型
- 需要传感器数据

**实现方法**:
```python
# 1. 获取预测速度
predicted_speeds = model.predict(current_speeds)

# 2. 插值到全路网
interpolator = SpeedInterpolator(mapper)
edge_speeds = interpolator.create_speed_field(predicted_speeds)

# 3. 修改路网图，添加预测速度
for (u, v, key), speed in edge_speeds.items():
    road_graph.G.edges[u, v, key]['predicted_speed'] = speed

# 4. 修改get_edge_speed方法使用预测速度
def get_edge_speed(self, u, v, key=0):
    # 优先使用预测速度
    if 'predicted_speed' in self.G.edges[u, v, key]:
        return self.G.edges[u, v, key]['predicted_speed']
    # 回退到OSM限速
    return self._parse_speed(self.G.edges[u, v, key].get("maxspeed", 60))
```

### 模式3: 实时速度模式 (高级)

**使用场景**: 实时导航系统

**速度来源**: 实时交通数据API

**优点**:
- 反映当前实时状况
- 动态规避拥堵
- 最准确的通行时间

**缺点**:
- 需要实时数据源
- 计算开销大
- 可能不稳定

**数据源示例**:
- Google Maps Traffic API
- Here Technologies Traffic API
- TomTom Traffic API

## 🎯 速度约束的优先级

建议的速度约束优先级：

```
1. 实时速度 (如果有) ↓
2. 预测速度 (如果有) ↓
3. OSM限速 (默认)
```

**实现示例**:
```python
def get_effective_speed(self, u, v, key=0):
    """获取有效速度（优先级：实时 > 预测 > OSM限速）"""
    edge_data = self.G.edges[u, v, key]
    
    # 1. 实时速度（最高优先级）
    if 'realtime_speed' in edge_data:
        return edge_data['realtime_speed']
    
    # 2. 预测速度（次优先级）
    if 'predicted_speed' in edge_data:
        return edge_data['predicted_speed']
    
    # 3. OSM限速（默认）
    if 'maxspeed' in edge_data:
        return self._parse_speed(edge_data['maxspeed'])
    
    # 4. 全局默认值
    return 60.0
```

## 📈 速度约束的优化策略

### 策略1: 分层速度约束

**思路**: 不同类型道路使用不同速度来源

```python
def get_speed_with_heuristics(self, u, v, key=0):
    edge_data = self.G.edges[u, v, key]
    highway_type = edge_data.get('highway', '')
    
    # 主干道：使用预测速度
    if highway_type in ['motorway', 'trunk', 'primary']:
        return edge_data.get('predicted_speed', 
                           self._parse_speed(edge_data.get('maxspeed', 80)))
    
    # 普通道路：使用OSM限速
    else:
        return self._parse_speed(edge_data.get('maxspeed', 50))
```

### 策略2: 时间段速度约束

**思路**: 根据不同时间段调整速度

```python
def get_speed_by_time(self, u, v, key=0, hour=None):
    """根据时间段获取速度"""
    edge_data = self.G.edges[u, v, key]
    base_speed = self._parse_speed(edge_data.get('maxspeed', 60))
    
    if hour is None:
        from datetime import datetime
        hour = datetime.now().hour
    
    # 早高峰 (7-9点): 速度降低到60%
    if 7 <= hour <= 9:
        return base_speed * 0.6
    
    # 晚高峰 (17-19点): 速度降低到50%
    elif 17 <= hour <= 19:
        return base_speed * 0.5
    
    # 其他时间: 正常速度
    else:
        return base_speed
```

### 策略3: 拥堵系数调整

**思路**: 使用历史拥堵系数调整速度

```python
def get_speed_with_congestion(self, u, v, key=0):
    """考虑拥堵系数的速度"""
    edge_data = self.G.edges[u, v, key]
    base_speed = self._parse_speed(edge_data.get('maxspeed', 60))
    
    # 获取该道路的拥堵系数（0.0-1.0，1.0表示无拥堵）
    congestion_factor = edge_data.get('congestion_factor', 0.8)
    
    return base_speed * congestion_factor
```

### 示例3: 动态权重更新

```python
# 创建自定义路由器
class TrafficAwareRouter(AStarRouter):
    def __init__(self, graph, speed_field):
        super().__init__(graph)
        self.speed_field = speed_field  # 'predicted_speed' or 'realtime_speed'
    
    def find_path(self, start_node, end_node, weight_type="time"):
        # 使用动态速度计算权重
        for u, v, key in self.graph.G.edges(keys=True):
            edge_data = self.graph.G.edges[u, v, key]
            
            # 获取动态速度
            if self.speed_field in edge_data:
                speed = edge_data[self.speed_field]
            else:
                speed = edge_data.get('maxspeed', 60)
            
            # 计算时间权重
            length = edge_data.get('length', 0)
            time_weight = length / (speed * 1000 / 3600)
            edge_data['dynamic_time_weight'] = time_weight
        
        # 使用动态权重搜索
        return super().find_path(start_node, end_node, 'dynamic_time')
```

## 🎯 总结

### 当前系统的速度约束机制

1. **基础层**: OSM的maxspeed属性（静态限速）
