# 路径规划算法比对分析功能说明

## 功能概述

新增的路径规划算法比对分析功能支持从**时间**和**距离**两个维度对比不同路径规划算法的性能表现。

## 支持的算法

1. **A* 算法** - 绿色路径
   - 启发式搜索算法
   - 结合实际路径成本和启发式估计
   - 通常在速度和路径质量之间达到良好平衡

2. **Dijkstra 算法** - 蓝色路径
   - 经典的最短路径算法
   - 保证找到最优解
   - 计算量较大，但结果可靠

3. **贪心算法** - 橙色路径
   - 基于启发式的快速搜索
   - 每次选择距离终点最近的邻居
   - 计算速度快，但可能不是最优路径

## 使用方法

### 访问比对分析页面

1. 启动后端服务：
   ```bash
   cd D:\D12_TGCN_Planning
   python backend/app.py
   ```

2. 在浏览器中打开主页：`http://localhost:5000`

3. 点击主页上的 **"📊 算法比对分析 →"** 按钮进入比对页面

### 操作步骤

1. **选择起点**
   - 点击地图上的任意位置设置起点（绿色标记）
   - 必须在蓝色虚线矩形框（路网覆盖范围）内

2. **选择终点**
   - 再次点击地图设置终点（红色标记）
   - 同样需要在路网覆盖范围内

3. **选择算法**
   - 勾选要比对的算法（至少选择一个）
   - 可以同时选择多个算法进行对比

4. **选择优化目标**
   - **最短时间**：优先选择行驶时间最短的路径
   - **最短距离**：优先选择物理距离最短的路径

5. **开始比对**
   - 点击 **"开始比对分析"** 按钮
   - 等待计算完成

## 结果展示

### 1. 统计表格
显示各算法的详细对比数据：
- **距离**：路径的总长度
- **时间**：预计行驶时间
- **计算耗时**：算法运行时间
- 最佳结果会用 ✓ 标记并高亮显示

### 2. 图表分析
- **距离对比图**：柱状图直观展示各算法的路径长度差异
- **时间对比图**：柱状图展示各算法的预计行驶时间差异

### 3. 地图可视化
- 在地图上同时绘制多条路径
- 每个算法使用不同颜色
- 可以直观对比路径走向差异

### 4. 分析结论
系统自动生成的智能分析：
- 📍 最短距离的算法
- ⏱️ 最短时间的算法
- ⚡ 平均计算耗时
- 💡 综合建议

## 应用场景

### 1. 算法性能评估
- 对比不同算法在同一路径上的表现
- 评估算法的计算效率
- 选择最适合特定场景的算法

- 研究不同算法生成的路径差异
- 分析算法在不同优化目标下的表现
- 为算法改进提供参考

### 3. 教学演示
- 直观展示不同路径规划算法的工作原理
- 对比算法优缺点
- 理解启发式搜索的作用

## 注意事项

1. **路网范围**
   - 起点和终点必须在PeMS D12路网覆盖范围内
   - Orange County, CA区域
   - 纬度：33.40°N ~ 33.94°N
   - 经度：118.09°W ~ 117.55°W

2. **计算时间**
   - 复杂路径可能需要较长计算时间
   - 建议选择中等距离的测试路径
   - 避免选择过于接近或过于遥远的起终点

3. **算法特点**
   - A*算法：推荐用于大多数场景
   - Dijkstra算法：适合需要最优解的场景
   - 贪心算法：适合快速估算的场景

## 扩展性

系统采用模块化设计，可以轻松添加新的路径规划算法：

1. 在 `backend/routing/` 下创建新的算法文件
2. 在 `RouterService` 中注册新算法
3. 在前端添加对应的选项
